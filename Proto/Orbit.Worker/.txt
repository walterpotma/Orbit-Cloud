O projeto Orbit.Calculo é o coração financeiro da plataforma. Já que você decidiu que o Agente de IA será em TypeScript, faz todo o sentido o motor de cálculo também ser em TS.Isso cria uma "stack de serviços auxiliares" homogênea, facilitando o compartilhamento de interfaces de tipos entre o Agente, o Cálculo e o Front-end (Next.js).1. Nomes Sugeridos (Naming)Para fugir do óbvio "Calculo", aqui estão nomes que passam a ideia de precisão, fluxo e nuvem:Orbit.Quasar: (Meu favorito) Quasares são os objetos mais brilhantes e energéticos do universo. Passa a ideia de um motor potente processando dados massivos.Orbit.Pulse: Como se fosse o "batimento cardíaco" do consumo, pulsando a cada minuto para registrar o gasto.Orbit.Flux: Focado no fluxo constante de créditos e débitos em tempo real.Orbit.Meter: Simples e direto, como um hidrômetro ou medidor de energia (Smart Meter).2. Planejamento Técnico (O que ele faz?)Como decidimos que o cálculo será por minuto (com aquele "flush" diário para economizar o banco no Hayom), o Orbit.Quasar (ou o nome que escolher) terá dois modos de operação:A. O Coletor (The Harvester)Frequência: Roda a cada 60 segundos.Ação: Consulta a sua API principal ou direto o Metrics Server do Kubernetes.Lógica: "O usuário X tem 2 Pods de 256MB rodando agora? Então ele gastou $R\$\ 0,0001$ neste minuto".Armazenamento: Salva no Redis (In-memory) para ser extremamente rápido e não fritar seu SSD com escritas constantes.B. O Consolidador (The Flusher)Frequência: Uma vez por dia (meia-noite).Ação: Soma todos os "minutos" do Redis.Lógica: "Ontem o Walter gastou um total de $R\$\ 1,45$".Persistência: Salva o valor final no PostgreSQL (seu banco oficial) e limpa o Redis para o dia seguinte.3. Por que TypeScript para o Cálculo?Manipulação de JSON: Os retornos do Prometheus/Kubernetes são JSONs densos. O TS com zod ou class-transformer lida com isso de forma muito elegante.Bibliotecas de Precisão: Para billing, você não pode usar number puro do JS (devido a erros de arredondamento de ponto flutuante). Em TS, você usará a lib big.js ou decimal.js para garantir que $0.1 + 0.2$ seja exatamente $0.300000$.Cron Jobs: O pacote node-cron ou bullmq (se quiser algo mais robusto) funciona perfeitamente para agendar essas tarefas.4. Integração com o EcossistemaServiçoResponsabilidadeOrbit.Api (.NET)Gerencia os planos e limites (Quota).Orbit.Quasar (TS)Lê o uso real, aplica a tarifa e gera o "balde" de gastos.Orbit.Agent (TS)Avisa se o saldo do usuário está acabando (baseado no Quasar).💡 Dica de "Arquitetura Pro"No seu servidor Xeon, como você está rodando tudo via Docker/Kubernetes, você pode usar o Redis como uma "ponte". O Orbit.Quasar escreve o gasto lá e o seu Dashboard Next.js lê do Redis via API para mostrar aquele gráfico que "sobe em tempo real". O usuário pira quando vê o centavo subindo enquanto o app dele está online!Walter, o que você acha do nome Orbit.Quasar?
